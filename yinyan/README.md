# yinyan 模块代码规划

目标：把“引言与表示学习”的核心概念做成可运行的小实验，尤其是传统机器学习、表示学习、深度学习三类方法差异。

## 建议新增文件与内容

| 文件名 | 应该写的代码内容 |
|---|---|
| `01_perceptron_xor_demo.py` | 感知机在 XOR 上失败的可视化实验，输出决策边界图和误分类点。 |
| `02_feature_engineering_for_xor.py` | 手工加入交叉特征（如 `x1*x2`）后用线性分类器解决 XOR。 |
| `03_mlp_xor_success.py` | 最小 MLP（1 个隐藏层）解决 XOR，对比训练前后准确率。 |
| `04_activation_compare.py` | Sigmoid/Tanh/ReLU 在同一任务上的收敛曲线与梯度分布对比。 |
| `05_backprop_from_scratch.py` | 纯 `numpy` 手写前向和反向传播，展示链式法则在代码中的对应。 |
| `06_representation_probe.py` | 提取不同层表示，做 PCA/t-SNE 可视化，观察“层越深越可分”。 |
| `07_history_timeline.ipynb` | 梳理感知机、反向传播、预训练、现代深度学习里程碑并配实验链接。 |
| `TraditionSignifiesDepth.ipynb` | 保留现有文件，建议改成“总览 notebook”，串起上述脚本的实验结论。 |

## 最小完成顺序

1. 先做 `01/02/03`，把“为什么要深度学习”讲清楚。  
2. 再做 `04/05`，解释“为什么深度网络能训练”。  
3. 最后做 `06/07`，补抽象与历史脉络。

## 完成标准

- 至少一张图能直观看到：线性模型与 MLP 的能力边界差异。  
- 至少一个脚本明确展示：表示在不同层的可分性变化。  
- 每个脚本都可以通过统一命令行参数运行（建议后续加 `--seed`）。
